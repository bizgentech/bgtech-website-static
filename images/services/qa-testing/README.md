# QA Services & Testing Images

This folder contains images for the QA Services & Testing page and related content.

## Image Guidelines

### Hero/Banner Images
- **Recommended Size:** 1920x1080px or 1600x900px
- **Format:** JPG (optimized) or WebP
- **Max File Size:** 500KB
- **Content:** QA engineers testing, automated test suites, bug tracking systems, testing environments

### Feature/Section Images
- **Recommended Size:** 800x600px or 1200x800px
- **Format:** JPG or WebP
- **Max File Size:** 300KB
- **Content:** Test automation, browser testing, mobile device testing, performance monitoring, security testing

### Testing Process Diagrams
- **Recommended Size:** 1200x800px
- **Format:** PNG or SVG
- **Content:** Testing workflows, QA processes, bug lifecycle diagrams

### Tool Screenshots
- **Recommended Size:** 1440x900px
- **Format:** PNG
- **Content:** Testing tools, automation frameworks, bug tracking interfaces

## Naming Convention

**Required File:**
- `hero.jpg` - Main hero image for the service page (1920x1080px, 300-500KB)

**Additional Images:**
Use descriptive, lowercase names with hyphens:
- `automated-testing.jpg`
- `mobile-device-testing.jpg`
- `performance-monitoring.jpg`
- `security-testing.jpg`
- `test-automation-suite.png`
- `feature-{name}.jpg` - For feature sections

## Current Usage

Images from this folder are used on:
- `/services/qa-testing` - Service detail page
  - `hero.jpg` - **REQUIRED** - Main hero background image
  - Testing process and tool images (optional)

**Note:** Home page carousel uses separate images from `/images/hero/`
